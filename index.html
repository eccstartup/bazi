💡 支付宝 AI收 (x402) 交互规范

1

Agent 发起数据查询

Agent 向 POST /v1/bazi/query 发送排盘请求。首次请求因不含凭证，服务端将被拦截。

2

服务端返回 402 账单挑战

服务端响应 HTTP 402 Payment Required 并返回 Payment-Needed 头部（包含 Base64 编码的嵌套账单与签名）。

3

用户支付宝 AI 钱包扣款

Agent 解析该账单头部，通过本地钱包插件（如 alipay-pay-for-402-service）触发扣款，获得 Payment-Proof 密文凭证。

4

重试完成，交付排盘

Agent 带上 Payment-Proof 请求头重新发送 POST。服务端通过支付宝网关验证凭证无误，确认履约并返回 HTTP 200 八字大图。
